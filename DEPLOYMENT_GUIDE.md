# BookNest Deployment Guide – Running on Another Device

This guide explains how to set up and run the BookNest project on another computer.

---

## System Requirements

Before setting up BookNest, ensure your device has:

- **Node.js** (v16 or higher) – [Download](https://nodejs.org/)
- **npm** (v8 or higher) – Comes with Node.js
- **Git** (optional, for cloning) – [Download](https://git-scm.com/)
- **~2GB** disk space for dependencies
- **Windows, macOS, or Linux**

**Verify installation:**
```bash
node --version    # Should show v16+
npm --version     # Should show v8+
```

---

## Option 1: Using Git (Recommended)

### Step 1: Clone the Repository
```bash
git clone https://github.com/your-username/booknest.git
cd booknest
```

### Step 2: Install Dependencies
```bash
npm run install:all
```

This command:
- Installs root dependencies (`razorpay`)
- Navigates to `frontend/` and installs frontend dependencies

---

## Option 2: Manual Transfer

### Step 1: Copy Project Files
Transfer the entire `booknest/` folder to the target device using:
- **USB Drive** – Copy folder to USB and transfer
- **Cloud Storage** – Upload to Google Drive, Dropbox, OneDrive, etc.
- **Email/File Transfer** – Use any file transfer service
- **Network Share** – Copy via shared network folder

**⚠️ Skip these folders** (they'll be recreated):
- `node_modules/` (regenerated by npm install)
- `.git/` (if not using Git)
- `.vercel/` (platform-specific)

### Step 2: Extract and Install
```bash
cd booknest
npm run install:all
```

---

## Environment Configuration

### Step 1: Copy Environment Variables
```bash
# The .env file already exists with demo credentials
# But you can customize it if needed
cp .env.example .env
```

### Step 2: Update .env (Optional)
Edit `.env` with your own values:

```env
# ── Razorpay API Keys ─────────────────────────────────────
# Get from https://dashboard.razorpay.com > Settings > API Keys
RAZORPAY_KEY_ID=rzp_test_YOUR_KEY_HERE
RAZORPAY_KEY_SECRET=YOUR_SECRET_HERE
VITE_RAZORPAY_KEY_ID=rzp_test_YOUR_KEY_HERE

# ── Admin Auth ─────────────────────────────────────────────
AUTH_ADMIN_EMAIL=admin@booknest.local
AUTH_ADMIN_PASSWORD=BookNest@2026
AUTH_USER_EMAIL=reader@booknest.local
AUTH_USER_PASSWORD=Reader@2026

# ── JWT Secret (generate a random long string) ─────────────
JWT_SECRET=your-super-secret-key-change-this
```

**Default Demo Credentials:**
| Role | Email | Password |
|------|-------|----------|
| Admin | `admin@booknest.local` | `BookNest@2026` |
| Customer | `reader@booknest.local` | `Reader@2026` |

## Running the Project

### Option A: Run Both Servers Automatically (Recommended for Development)

**Terminal 1 – Run both API and Frontend:**
```bash
cd booknest
npm run dev
```

This runs the frontend (Vite) in dev mode automatically.

**Terminal 2 – Run API Server in background:**
```bash
cd booknest
node scripts/local-api-server.cjs
```

### Option B: Run Servers Separately (More Control)

**Terminal 1 – Start API Server:**
```bash
cd booknest
node scripts/local-api-server.cjs
```
Expected output:
```
BookNest API listening on http://127.0.0.1:3001
```

**Terminal 2 – Start Frontend Server:**
```bash
cd booknest/frontend
npm run dev
```
Expected output:
```
Local:   http://localhost:5173/
```

### Access the Application

Open your browser and go to:
- **Frontend:** http://localhost:5173/
- **API:** http://127.0.0.1:3001/ (backend only, not browsable)

## Stopping the Servers

### On Windows/macOS/Linux:
1. **For Terminal running server:** Press `Ctrl + C`
2. **Kill lingering process (if needed):**
   ```bash
   # Windows PowerShell
   Get-Process node | Stop-Process -Force
   
   # macOS/Linux
   pkill -f node
   ```

## Troubleshooting

### Issue: "npm: command not found"
**Solution:** Node.js not installed. Download from https://nodejs.org/

### Issue: "Module not found: razorpay"
**Solution:** Run `npm run install:all` to install dependencies

### Issue: "Port 3001 already in use"
**Solution:** Stop other services or modify `local-api-server.cjs`:
```bash
# Check what's using port 3001
# Windows:
netstat -ano | findstr :3001

# macOS/Linux:
lsof -i :3001
```

### Issue: "EACCES: permission denied" (macOS/Linux)
**Solution:** Use sudo or fix npm permissions:
```bash
sudo npm install
# Or fix npm without sudo:
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH
```

### Issue: Frontend not connecting to API
**Solution:** Ensure API server is running on port 3001. Check `frontend/src/App.jsx` for API base URL configuration.

### Issue: "Cannot find module" errors
**Solution:** Delete `node_modules/` and reinstall:
```bash
rm -rf node_modules/
npm run install:all
```

## Development Tips

### Run Production Build (Optional)
```bash
cd booknest/frontend
npm run build
```
Output goes to `frontend/dist/` for deployment to Vercel or other services.

### View API Mock Data

- Demo orders

```env
AUTH_ADMIN_EMAIL=your_admin_email@domain.com

```

## Production Deployment

For cloud deployment (Vercel, AWS, Heroku, etc.):

1. **Build Frontend:**
   ```bash
   cd frontend
   npm run build
   ```

2. **Deploy to Vercel** (recommended):
   - Connect GitHub repo to Vercel
   - Set environment variables in Vercel dashboard
   - Auto-deploys on git push

3. **Deploy to Other Platforms:**
   - Use `npm run build` output
   - Follow platform-specific guides

## Support

For issues or questions:
1. Check `.env` configuration
2. Verify Node.js version: `node --version`
3. Try clearing cache: `npm cache clean --force`
4. Check API logs in terminal for errors
5. Restart both servers: Stop with `Ctrl+C` and restart
